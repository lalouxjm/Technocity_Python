Q1 Given a sorted array, find two numbers that add up to a target. You must do it in O(n) without extra memory. ✓

Answer

Two Pointers

The array is sorted and we want a pair. Start left=0, right=end. If sum too small → move left right. If too big → move right left. O(n) time, O(1) space. Hash map would also give O(n) but uses O(n) extra memory — the constraint rules it out.

Code

def two_sum_sorted(arr, target):
  left, right = 0, len(arr) - 1
  while left < right:
    s = arr[left] + arr[right]
    if   s == target: return [left, right]
    elif s <  target: left += 1
    else:             right -= 1
  return []
Q2 Find the length of the longest substring without repeating characters. E.g. "abcabcbb" → 3 ("abc"). ✓

Answer

Sliding Window

Variable-size window: expand right to add characters, shrink left when a duplicate enters the window. Use a set to track window contents. Classic variable sliding window — "longest substring with constraint" is the canonical signal.

Code

def length_of_longest(s):
  seen = set()
  left = best = 0
  for right, ch in enumerate(s):
    while ch in seen:
      seen.discard(s[left]); left += 1
    seen.add(ch)
    best = max(best, right - left + 1)
  return best

length_of_longest("abcabcbb")  # 3
Q3 You have a sorted array of 1 million elements. Find whether a value exists. The interviewer says: "think about efficiency." ✓

Answer

Binary Search

Sorted array + search = binary search. O(log n) means 1M elements requires at most 20 comparisons. A hash map would be O(1) per lookup but O(n) to build and O(n) memory — overkill if the array is already sorted and you're searching once.

Code

def search(arr, target):
  lo, hi = 0, len(arr) - 1
  while lo <= hi:
    mid = (lo + hi) // 2
    if   arr[mid] == target: return True
    elif arr[mid] <  target: lo = mid + 1
    else:                    hi = mid - 1
  return False
# 1M elements → max 20 iterations (log₂1M ≈ 20)
Q4 Find all pairs in an unsorted array that sum to a target. The array has no special ordering. Optimize for time. ✓

Answer

Hash Map

The array is unsorted — two pointers require a sorted array. Hash map: for each number, check if target - num is already stored. O(n) time, O(n) space. Two pointers would require O(n log n) sort first.

Code

def two_sum(nums, target):
  seen = {}
  for i, n in enumerate(nums):
    complement = target - n
    if complement in seen:
      return [seen[complement], i]
    seen[n] = i
  return []

# Key insight: check complement BEFORE storing
# to avoid using the same element twice
Q5 Count the number of subarrays that sum to exactly k. The array has both positive and negative numbers. ✓

Answer

Prefix Sum + Hash Map

Sliding window only works with positive numbers (window shrinks meaningfully). With negatives, prefix sum + hash map is the key: count += seen[prefix - k] counts subarrays ending here that sum to k. O(n) time and space.

Code

def subarray_sum(nums, k):
  count = prefix = 0
  seen = {0: 1}   # prefix 0 seen once
  for n in nums:
    prefix += n
    count += seen.get(prefix - k, 0)
    seen[prefix] = seen.get(prefix, 0) + 1
  return count

subarray_sum([1, -1, 1, 1], 1)  # 3

Q6 Find the shortest path between two nodes in an unweighted graph. "Shortest" means fewest edges. ✓

Answer

BFS

BFS explores level by level — all nodes at distance 1, then distance 2, etc. The first time you reach the target, you've found the shortest path. DFS would find a path, not necessarily the shortest. Dijkstra is for weighted graphs.

Code

from collections import deque

def shortest_path(graph, src, dst):
  queue = deque([(src, 0)])
  visited = {src}
  while queue:
    node, dist = queue.popleft()
    if node == dst: return dist
    for nei in graph[node]:
      if nei not in visited:
        visited.add(nei)
        queue.append((nei, dist+1))
  return -1  # not reachable
Q7 Count the number of islands in a 2D grid of '1' (land) and '0' (water). Connected '1's (4-directionally) form one island. ✓

Answer

DFS

Classic DFS (or BFS — both work). When you find a '1', DFS into all connected cells and mark them visited. Each time you start a DFS from an unmarked '1', increment the island count. Union-Find also works but DFS is more intuitive here.

Code

def num_islands(grid):
  count = 0
  def dfs(r, c):
    if not (0<=r<len(grid) and 0<=c<len(grid[0])
            and grid[r][c]=='1'): return
    grid[r][c] = '0'  # sink it
    for dr,dc in [(1,0),(-1,0),(0,1),(0,-1)]:
      dfs(r+dr, c+dc)
  for r in range(len(grid)):
    for c in range(len(grid[0])):
      if grid[r][c]=='1': dfs(r,c); count+=1
  return count
Q8 Generate all permutations of a list of distinct integers. E.g. [1,2,3] → all 6 orderings. ✓

Answer

Backtracking

"Generate ALL …" is the backtracking signal. Build the permutation incrementally: choose a number, recurse, then unchoose (backtrack). The choose/recurse/unchoose template works for permutations, combinations, subsets, and constraint satisfaction.

Code

def permutations(nums):
  result = []
  def backtrack(current, remaining):
    if not remaining:
      result.append(current[:]); return
    for i in range(len(remaining)):
      current.append(remaining[i])
      backtrack(current, remaining[:i] + remaining[i+1:])
      current.pop()
  backtrack([], nums)
  return result
Q9 Detect if a linked list has a cycle. You must use O(1) extra memory — no hash set allowed. ✓

Answer

Fast & Slow Pointers

Floyd's cycle detection: fast moves 2 steps, slow moves 1. If a cycle exists, fast will "lap" slow and they meet. If no cycle, fast reaches null. This is O(1) space — the constraint makes this the only valid answer (hash set would be O(n)).

Code

def has_cycle(head):
  slow, fast = head, head
  while fast and fast.next:
    slow = slow.next
    fast = fast.next.next
    if slow is fast:
      return True   # met → cycle!
  return False

# Key: "fast laps slow" only happens in a cycle
# O(n) time, O(1) space ✓
Q10 You have a Binary Search Tree. Produce the values in ascending (sorted) order without any extra sorting step. ✓

Answer

Inorder traversal

The BST property guarantees: left subtree < node < right subtree. Inorder (Left → Node → Right) visits them in exactly this order, producing sorted output. This is a fundamental BST property — memorize it.

Code

def inorder(node, result=[]):
  if not node: return result
  inorder(node.left, result)   # Left
  result.append(node.val)      # Node
  inorder(node.right, result)  # Right
  return result

# BST inorder = sorted array
# Preorder  = Node L R (serialize)
# Postorder = L R Node (delete/height)

Q11 Count the number of distinct ways to climb n stairs, taking 1 or 2 steps at a time. E.g. n=4 → 5 ways. ✓

Answer

Dynamic Programming

"Count the number of ways" + overlapping subproblems = DP. ways(n) = ways(n-1) + ways(n-2) — Fibonacci pattern. Backtracking would find all paths (exponential), DP caches the count (linear).

Code

def climb_stairs(n):
  if n <= 2: return n
  prev, curr = 1, 2
  for _ in range(3, n + 1):
    prev, curr = curr, prev + curr
  return curr

# dp[i] = dp[i-1] + dp[i-2]  (Fibonacci!)
climb_stairs(4)  # 5
climb_stairs(5)  # 8
Q12 You have a list of meetings with start/end times. Find the minimum number of conference rooms needed to hold all of them simultaneously. ✓

Answer

Greedy + Heap

Sort meetings by start. Use a min-heap tracking end times of active meetings. For each meeting: if the earliest-ending room is free (heap top ≤ current start), reuse it (heapreplace). Otherwise, open a new room (heappush). The heap size = rooms needed.

Code

import heapq

def min_rooms(intervals):
  intervals.sort(key=lambda x: x[0])
  heap = []   # end times of active rooms
  for start, end in intervals:
    if heap and heap[0] <= start:
      heapq.heapreplace(heap, end)  # reuse
    else:
      heapq.heappush(heap, end)     # new room
  return len(heap)
Q13 Given [[1,3],[2,6],[8,10],[15,18]], merge all overlapping intervals. Output: [[1,6],[8,10],[15,18]]. ✓

Answer

Merge Intervals

The canonical merge intervals pattern: sort by start, keep a result list, extend the last interval's end if the current one overlaps (curr.start ≤ last.end), otherwise append. O(n log n) for the sort, O(n) for the merge pass.

Code

def merge(intervals):
  intervals.sort(key=lambda x: x[0])
  merged = [intervals[0]]
  for s, e in intervals[1:]:
    if s <= merged[-1][1]:   # overlap
      merged[-1][1] = max(merged[-1][1], e)
    else:
      merged.append([s, e])
  return merged
Q14 Find the K most frequent words in a list. Sorting all words would give O(n log n) — can you do better? ✓

Answer

Min-Heap of size K

Count frequencies with a dict, then maintain a min-heap of size K. Each push+pop is O(log k). Total: O(n log k) vs O(n log n) for full sort. When k is much smaller than n, the improvement is significant. The heap gives you the top K without sorting everything.

Code

import heapq
from collections import Counter

def top_k_frequent(words, k):
  counts = Counter(words)
  # min-heap by (-freq, word) for tiebreak
  heap = []
  for word, freq in counts.items():
    heapq.heappush(heap, (freq, word))
    if len(heap) > k:
      heapq.heappop(heap)
  return [w for _, w in sorted(heap, reverse=True)]
Q15 Given two strings, find the length of their longest common subsequence. E.g. "abcde" and "ace" → 3. ✓

Answer

Dynamic Programming (2D)

Classic DP: dp[i][j] = LCS of s1[:i] and s2[:j]. If characters match: dp[i][j] = dp[i-1][j-1] + 1. Otherwise: max(dp[i-1][j], dp[i][j-1]). Backtracking would be exponential; DP is O(m×n).

Code

def lcs(s1, s2):
  m, n = len(s1), len(s2)
  dp = [[0] * (n+1) for _ in range(m+1)]
  for i in range(1, m+1):
    for j in range(1, n+1):
      if s1[i-1] == s2[j-1]:
        dp[i][j] = dp[i-1][j-1] + 1
      else:
        dp[i][j] = max(dp[i-1][j], dp[i][j-1])
  return dp[m][n]

Q16 Given n nodes and a list of edges added one by one, determine after each addition whether all nodes are connected. Must be efficient for dynamic updates. ✓

Answer

Union-Find

BFS/DFS after each edge is O(V+E) per edge → O(n²) total. Union-Find merges components in near-O(1) amortized. Track the number of distinct components: start at n, decrement each time you union two different sets. When count=1, all connected.

Code

class UF:
  def __init__(self, n):
    self.p = list(range(n)); self.count = n
  def find(self, x):
    if self.p[x]!=x: self.p[x]=self.find(self.p[x])
    return self.p[x]
  def union(self, x, y):
    px,py = self.find(x), self.find(y)
    if px!=py: self.p[px]=py; self.count-=1
  def connected(self): return self.count == 1
Q17 For each day in a temperature list, find how many days until a warmer temperature. E.g. [73,74,75,71] → [1,1,0,0]. ✓

Answer

Monotonic Stack

"Next greater element" = monotonic stack. Maintain a stack of indices of days we haven't resolved yet (in decreasing temperature order). When a warmer day arrives, pop everything cooler and record the wait. O(n) — each index pushed and popped once.

Code

def daily_temps(temps):
  ans = [0] * len(temps)
  stack = []  # indices, decreasing temp
  for i, t in enumerate(temps):
    while stack and temps[stack[-1]] < t:
      j = stack.pop()
      ans[j] = i - j
    stack.append(i)
  return ans

daily_temps([73,74,75,71,69,72,76,73])
# [1, 1, 4, 2, 1, 1, 0, 0]
Q18 Build a search feature that instantly checks if any stored word starts with a given prefix. E.g. given words ["apple","app","banana"], startsWith("app") → True. ✓

Answer

Trie

Tries are built for prefix queries. Lookup is O(L) where L is the prefix length — independent of how many words are stored. A hash set would require iterating all words to check prefixes. Binary search works on full words but not arbitrary prefixes efficiently.

Code

class Trie:
  def __init__(self):
    self.root = {}

  def insert(self, word):
    node = self.root
    for ch in word:
      node = node.setdefault(ch, {})
    node["#"] = True    # end marker

  def starts_with(self, prefix):
    node = self.root
    for ch in prefix:
      if ch not in node: return False
      node = node[ch]
    return True
Q19 You have n courses. Some courses have prerequisites. Determine if it's possible to complete all courses (detect if there's a circular dependency). ✓

Answer

Topological Sort

Dependency ordering = topological sort. Use Kahn's algorithm (BFS-based): track in-degrees, start with 0-in-degree nodes, reduce neighbors' in-degrees. If all n courses are processed → no cycle → possible. If stuck before processing all → cycle detected.

Code

from collections import deque, defaultdict

def can_finish(n, prereqs):
  indeg = [0] * n; g = defaultdict(list)
  for a, b in prereqs:
    g[b].append(a); indeg[a] += 1
  q = deque(i for i in range(n) if indeg[i]==0)
  done = 0
  while q:
    u = q.popleft(); done += 1
    for v in g[u]:
      indeg[v] -= 1
      if indeg[v]==0: q.append(v)
  return done == n
Q20 A network has nodes and weighted edges (non-negative). Find the minimum time for a signal to reach all nodes from a given source. ✓

Answer

Dijkstra's Algorithm

Weighted graph + shortest path + non-negative weights = Dijkstra. BFS only works for unweighted (or unit-weight) graphs. Dijkstra uses a min-heap: always expand the closest unvisited node. Return the max distance to all nodes — if any unreachable, return -1.

Code

import heapq
from collections import defaultdict

def network_delay(times, n, src):
  g = defaultdict(list)
  for u,v,w in times: g[u].append((w,v))
  dist = {src:0}; heap = [(0,src)]
  while heap:
    d,u = heapq.heappop(heap)
    if d > dist.get(u,float('inf')): continue
    for w,v in g[u]:
      nd = d+w
      if nd < dist.get(v,float('inf')):
        dist[v]=nd; heapq.heappush(heap,(nd,v))
  return max(dist.values()) if len(dist)==n else -1
