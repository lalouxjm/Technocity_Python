Array & String patterns

The most common interview patterns — master these first.

Two Pointers

pattern

Two indices moving through a sequence — often from both ends — to avoid a nested loop. Reduces O(n²) brute-force to O(n).

O(n) time · O(1) space
Classic: two sum (sorted), palindrome check, container with most water, reverse in-place

When to use it

The array is sorted, or you're looking for a pair/triple that satisfies a condition. One pointer starts at the left, one at the right. Based on the result, move the one that can improve it.

Example — pair summing to target

def has_pair_sum(arr, target):
  left, right = 0, len(arr) - 1
  while left < right:
    s = arr[left] + arr[right]
    if s == target:   return True
    elif s < target: left += 1
    else:             right -= 1
  return False

# arr must be sorted!
has_pair_sum([1,3,5,7,9], 10)  # True (3+7)
Spot it when: sorted arrayfind pair/triplein-place reversepalindrome check

Sliding Window

pattern

A window of fixed or variable size slides over an array or string. Subtract what leaves, add what enters — avoids recomputing from scratch each step.

O(n) time · O(1) or O(k) space
Classic: max sum of k elements, longest substring without repeating chars, min size subarray sum

When to use it

The problem involves a contiguous subarray or substring. "Longest", "shortest", "maximum sum" of a sub-sequence are strong signals. Fixed-size: two pointers k apart. Variable-size: expand right, shrink left when condition breaks.

Example — max sum subarray of size k

def max_sum_k(nums, k):
  window = sum(nums[:k])
  best = window
  for i in range(k, len(nums)):
    window += nums[i] - nums[i - k]  # slide
    best = max(best, window)
  return best

max_sum_k([2,1,5,1,3,2], 3)  # 9  (5+1+3)
Spot it when: contiguous subarraysubstringwindow of size klongest / shortest

Binary Search

pattern + algorithm

Repeatedly halve the search space on a sorted array or any monotonic condition. Each step eliminates half the candidates.

O(log n) time · O(1) space
Classic: search in rotated array, first bad version, find peak element, search insert position

When to use it

The input is sorted, or the problem has a monotonic condition (true/false boundary). Also use when brute force is O(n) and you suspect O(log n) is possible. Think: "can I eliminate half the options at each step?"

Example — standard binary search

def binary_search(arr, target):
  lo, hi = 0, len(arr) - 1
  while lo <= hi:
    mid = (lo + hi) // 2
    if   arr[mid] == target: return mid
    elif arr[mid] <  target: lo = mid + 1
    else:                    hi = mid - 1
  return -1
Spot it when: sorted arrayO(log n) hintmonotonic condition"find first/last"

Hash Map / Hash Set

pattern

O(1) average lookups to count, deduplicate, or map relationships between elements. Trade memory for speed.

O(n) time · O(n) space
Classic: two sum, group anagrams, longest consecutive sequence, character frequency

When to use it

You need to check membership, count occurrences, or find a complement — and you need it fast. If you're writing x in list inside a loop, replace that list with a set or dict. Almost every medium problem uses this.

Example — group anagrams

from collections import defaultdict

def group_anagrams(words):
  groups = defaultdict(list)
  for w in words:
    key = tuple(sorted(w))   # canonical form
    groups[key].append(w)
  return list(groups.values())

group_anagrams(["eat","tea","tan","ate","nat"])
# [["eat","tea","ate"], ["tan","nat"]]
Spot it when: count occurrencesfind complementgroup by keyO(1) membership

Prefix Sum

pattern

Precompute cumulative sums once in O(n), then answer range sum queries in O(1) instead of recomputing each time.

O(n) build · O(1) query
Classic: range sum query, subarray sum equals k, product of array except self

When to use it

Multiple queries about sums of subarrays. Build prefix[i] = sum(arr[0..i]) once. Then sum(l, r) = prefix[r] - prefix[l-1] in O(1). Combine with a hash map to find subarrays with a given sum.

Example — subarray sum equals k

def subarray_sum(nums, k):
  count = 0
  prefix = 0
  seen = {0: 1}   # prefix_sum → frequency
  for n in nums:
    prefix += n
    count += seen.get(prefix - k, 0)
    seen[prefix] = seen.get(prefix, 0) + 1
  return count

subarray_sum([1,1,1], 2)  # 2
Spot it when: range sum queriessubarray sum = kmultiple queries on same array

Stack

pattern

LIFO structure to process elements in reverse order or track nested/contextual state. Essential for bracket problems, expression evaluation, and monotonic sequences.

O(n) time · O(n) space
Classic: valid parentheses, next greater element, min stack, evaluate expression

When to use it

You need to "remember what came before" or process things in reverse. Nested structures (brackets, HTML), "next greater element", and undo-style problems all scream stack. Use list.append() / list.pop() — both O(1).

Example — next greater element

def next_greater(nums):
  res = [-1] * len(nums)
  stack = []  # stores indices
  for i, n in enumerate(nums):
    while stack and nums[stack[-1]] < n:
      res[stack.pop()] = n
    stack.append(i)
  return res

next_greater([2,1,2,4,3])
# [4, 2, 4, -1, -1]
Spot it when: nested structurematching bracketsnext greater/smallerreverse processing

Graph & Tree patterns

Traversal strategies — the foundation of graph and tree problem solving.

Depth-First Search (DFS)

pattern + algorithm

Traverse as deep as possible before backtracking, using recursion or an explicit stack. Explores all paths fully before trying siblings.

O(V+E) time · O(V) space
Classic: number of islands, path sum, connected components, clone graph

When to use it

Explore all paths, detect cycles, find connected components, or solve maze/grid problems. Prefer DFS over BFS when you need to reach leaves or explore all possibilities. Use a visited set to avoid infinite loops.

Example — number of islands (grid DFS)

def num_islands(grid):
  count = 0
  def dfs(r, c):
    if r < 0 or r >= len(grid) or \
       c < 0 or c >= len(grid[0]) or grid[r][c] == "0":
      return
    grid[r][c] = "0"   # mark visited
    for dr, dc in [(1,0),(-1,0),(0,1),(0,-1)]:
      dfs(r+dr, c+dc)
  for r in range(len(grid)):
    for c in range(len(grid[0])):
      if grid[r][c] == "1": dfs(r,c); count+=1
  return count
Spot it when: explore all pathsconnected componentsgrid traversaldetect cycle

Breadth-First Search (BFS)

pattern + algorithm

Traverse level by level using a queue. Guarantees the shortest path in unweighted graphs because it explores all nodes at distance d before d+1.

O(V+E) time · O(V) space
Classic: shortest path, level-order traversal, word ladder, rotting oranges

When to use it

You need the shortest path or minimum steps in an unweighted graph. BFS processes nodes level by level — the first time you reach the target, you have the shortest path. Always use a deque, never list.pop(0).

Example — shortest path in grid

from collections import deque

def shortest_path(grid):
  rows, cols = len(grid), len(grid[0])
  queue = deque([(0, 0, 1)])   # r, c, steps
  visited = {(0, 0)}
  while queue:
    r, c, steps = queue.popleft()
    if r == rows-1 and c == cols-1: return steps
    for dr, dc in [(1,0),(-1,0),(0,1),(0,-1)]:
      nr, nc = r+dr, c+dc
      if 0<=nr<rows and 0<=nc<cols and (nr,nc) not in visited:
        visited.add((nr,nc)); queue.append((nr,nc,steps+1))
Spot it when: shortest pathmin stepslevel by levelunweighted graph

Tree Traversals

pattern + algorithm

Inorder, preorder, postorder — each reveals different structural properties of a binary tree. Inorder on BST gives sorted output.

O(n) time · O(h) space
Classic: validate BST, lowest common ancestor, serialize/deserialize tree, diameter

The three orders

Inorder (L → Node → R): gives sorted values in a BST.
Preorder (Node → L → R): serialize a tree, copy structure.
Postorder (L → R → Node): process children before parent — good for deletion, computing subtree results.

Example — inorder (recursive)

def inorder(node, result=[]):
  if not node: return
  inorder(node.left, result)
  result.append(node.val)
  inorder(node.right, result)
  return result

# Postorder — process subtrees first
def height(node):
  if not node: return 0
  return 1 + max(height(node.left),
                  height(node.right))
Spot it when: binary treeBST propertysubtree computationserialize tree

Recursion & Backtracking

pattern

Explore all possibilities by building candidates incrementally and abandoning (backtracking) paths that fail a constraint early.

O(n!) worst · varies
Classic: permutations, subsets, combination sum, N-Queens, word search

When to use it

Generate all combinations, permutations, or subsets. The template: choose → recurse → unchoose. Prune early when a partial candidate already violates constraints. This makes it vastly faster than brute force in practice.

Example — all subsets

def subsets(nums):
  result = []
  def backtrack(start, current):
    result.append(current[:])  # snapshot
    for i in range(start, len(nums)):
      current.append(nums[i])    # choose
      backtrack(i + 1, current)  # recurse
      current.pop()              # unchoose
  backtrack(0, [])
  return result

subsets([1,2,3])
# [[], [1], [1,2], [1,2,3], [1,3], [2], [2,3], [3]]
Spot it when: generate all …permutationscombinationsconstraint satisfaction

Fast & Slow Pointers

pattern

Two pointers moving at different speeds through a linked list. The fast pointer moves 2 steps, the slow 1. When fast hits the end, slow is at the middle.

O(n) time · O(1) space
Classic: detect cycle, find middle node, find cycle entry point, happy number

When to use it

Linked list problems where you can't index directly. Cycle detection: if there's a cycle, fast and slow will eventually meet. Finding the middle: when fast reaches the end, slow is exactly at the middle.

Example — detect cycle in linked list

def has_cycle(head):
  slow, fast = head, head
  while fast and fast.next:
    slow = slow.next
    fast = fast.next.next
    if slow is fast:
      return True    # they met → cycle!
  return False

# Find middle: when fast ends, slow = middle
def find_middle(head):
  slow, fast = head, head
  while fast and fast.next:
    slow = slow.next; fast = fast.next.next
  return slow
Spot it when: linked listdetect cyclefind middleO(1) space constraint

Dynamic Programming & Greedy

Optimization patterns — break problems into subproblems or make locally optimal choices.

Dynamic Programming

pattern

Break a problem into overlapping subproblems; cache results to avoid redundant computation. Two approaches: top-down (memoization) and bottom-up (tabulation).

O(n) or O(n²) typical
Classic: climbing stairs, 0/1 knapsack, longest common subsequence, coin change

When to use it

The problem asks for a count, min, or max over all possibilities, and subproblems overlap. Check: "Can I define this answer in terms of smaller versions of the same problem?" If yes — DP. If no overlapping subproblems — try greedy or backtracking.

Example — coin change (min coins)

def coin_change(coins, amount):
  dp = [float("inf")] * (amount + 1)
  dp[0] = 0         # base case
  for a in range(1, amount + 1):
    for c in coins:
      if c <= a:
        dp[a] = min(dp[a], dp[a - c] + 1)
  return dp[amount] if dp[amount] != float("inf") else -1

coin_change([1,5,6,9], 11)  # 2  (5+6)
Spot it when: count/min/max over all waysoverlapping subproblems"how many ways"

Greedy

pattern

Make the locally optimal choice at each step — works when local optima lead to a global optimum. No backtracking needed.

O(n log n) typical (sort first)
Classic: jump game, gas station, activity selection, assign cookies

When to use it

Problems where a locally optimal choice never needs to be reconsidered. Usually involves sorting first. Greedy is faster than DP but only valid when the "greedy choice property" holds — you must prove or intuit it for each problem.

Example — jump game (can you reach end?)

def can_jump(nums):
  max_reach = 0
  for i, n in enumerate(nums):
    if i > max_reach:
      return False    # stuck — can't get here
    max_reach = max(max_reach, i + n)
  return True

# Greedily track the farthest we can reach
can_jump([2,3,1,1,4])  # True
can_jump([3,2,1,0,4])  # False
Spot it when: locally optimal = globally optimalschedulinginterval problems

Merge Intervals

pattern

Sort intervals by start time, then merge overlapping ones in a single linear pass. A specific and very common interview pattern.

O(n log n) sort + O(n) merge
Classic: merge intervals, insert interval, meeting rooms I & II, non-overlapping intervals

When to use it

Any problem involving overlapping ranges, scheduling, or time intervals. Key insight: after sorting by start, two intervals overlap if curr.start <= prev.end. Merge by extending the end to max(prev.end, curr.end).

Example — merge overlapping intervals

def merge(intervals):
  intervals.sort(key=lambda x: x[0])
  merged = [intervals[0]]
  for start, end in intervals[1:]:
    if start <= merged[-1][1]:
      merged[-1][1] = max(merged[-1][1], end)
    else:
      merged.append([start, end])
  return merged

merge([[1,3],[2,6],[8,10],[15,18]])
# [[1,6],[8,10],[15,18]]
Spot it when: time intervalsschedulingoverlapping rangesmeeting rooms

Heap / Top K Elements

pattern

Maintain fast access to the current minimum or maximum of a dynamic set. A min-heap of size K efficiently tracks the K largest elements — without full sorting.

O(n log k) time · O(k) space
Classic: K largest elements, merge K sorted lists, median from stream, task scheduler

When to use it

The problem mentions "top K", "K largest/smallest", or "median". Python's heapq is a min-heap. To simulate a max-heap, negate values. Pushing onto a heap of size K and popping the min gives you the K largest in O(n log k).

Example — K largest elements

import heapq

def k_largest(nums, k):
  heap = []
  for n in nums:
    heapq.heappush(heap, n)
    if len(heap) > k:
      heapq.heappop(heap)  # remove smallest
  return heap   # contains k largest

k_largest([3,1,5,12,2,11], 3)
# [5, 11, 12] (heap, unsorted)
Spot it when: top KK largest/smallestmedian from streampriority queue

Advanced patterns

Less common in standard interviews — essential for senior roles and competitive programming.

Union-Find (DSU)

pattern + algorithm

Tracks a partition of elements into disjoint sets with near-O(1) union and find. Uses path compression and union by rank.

O(α(n)) ≈ O(1) amortized
Classic: number of connected components, redundant connection, accounts merge

When to use it

Dynamically connecting and grouping elements. "Are A and B in the same group?" or "Connect A and B". Alternative to BFS/DFS for connected components — often cleaner. Path compression makes repeated operations nearly O(1).

Example — Union-Find template

class UnionFind:
  def __init__(self, n):
    self.parent = list(range(n))
    self.rank = [0] * n

  def find(self, x):
    if self.parent[x] != x:
      self.parent[x] = self.find(self.parent[x])
    return self.parent[x]   # path compression

  def union(self, x, y):
    px, py = self.find(x), self.find(y)
    if self.rank[px] < self.rank[py]: px, py = py, px
    self.parent[py] = px
Spot it when: dynamic connectivityconnected components"same group?"

Monotonic Stack

pattern

A stack kept strictly increasing or decreasing to efficiently find the next/previous greater or smaller element in O(n) instead of O(n²).

O(n) time · O(n) space
Classic: largest rectangle in histogram, daily temperatures, trapping rain water

When to use it

You need "next greater element", "previous smaller element", or area/volume calculations involving bars (histogram, rain water). Maintain a stack in monotonic order — when a new element breaks the order, pop and process.

Example — daily temperatures

def daily_temps(temps):
  ans = [0] * len(temps)
  stack = []  # indices, decreasing temps
  for i, t in enumerate(temps):
    while stack and temps[stack[-1]] < t:
      j = stack.pop()
      ans[j] = i - j    # days to wait
    stack.append(i)
  return ans

daily_temps([73,74,75,71,69,72,76,73])
# [1, 1, 4, 2, 1, 1, 0, 0]
Spot it when: next greater elementhistogramtrapping waterspan problems

Trie (Prefix Tree)

pattern

Tree where each node is a character — enables fast prefix-based search and insertion. Each path from root to leaf spells a word.

O(L) insert/search (L = word length)
Classic: implement trie, word search II, autocomplete, replace words

When to use it

Problems involving prefix lookups, autocomplete, or searching a set of words. More efficient than a hash set when you need prefix queries (e.g., "does any word start with 'app'?").

Example — Trie insert & search

class TrieNode:
  def __init__(self):
    self.children = {}; self.end = False

class Trie:
  def __init__(self): self.root = TrieNode()

  def insert(self, word):
    node = self.root
    for ch in word:
      node = node.children.setdefault(ch, TrieNode())
    node.end = True

  def starts_with(self, prefix):
    node = self.root
    for ch in prefix:
      if ch not in node.children: return False
      node = node.children[ch]
    return True
Spot it when: prefix searchautocompleteword dictionarystarts with

Topological Sort

pattern + algorithmless common

Produce a linear ordering of a DAG (Directed Acyclic Graph) that respects all directed dependencies. Detect circular dependencies.

O(V+E) time
Classic: course schedule, build order, alien dictionary, task dependencies

When to use it

"Can you complete all tasks given these dependencies?" = topological sort. Build an in-degree map, start with 0-in-degree nodes, and process via BFS (Kahn's algorithm). If not all nodes are processed, a cycle exists.

Example — course schedule (Kahn's algorithm)

from collections import deque, defaultdict

def can_finish(n, prerequisites):
  indegree = [0] * n
  graph = defaultdict(list)
  for a, b in prerequisites:
    graph[b].append(a); indegree[a] += 1
  queue = deque(i for i in range(n) if indegree[i] == 0)
  done = 0
  while queue:
    node = queue.popleft(); done += 1
    for nei in graph[node]:
      indegree[nei] -= 1
      if indegree[nei] == 0: queue.append(nei)
  return done == n
Spot it when: dependenciesDAG orderingcourse scheduledetect cycle in DAG

Dijkstra's Algorithm

pattern + algorithmless common

Shortest path from a source in a weighted graph using a priority queue. Greedy: always expand the currently cheapest node.

O((V+E) log V)
Classic: network delay time, cheapest flights within k stops, path with min effort

When to use it

Shortest path in a weighted graph with non-negative edges. BFS finds shortest path in unweighted graphs; Dijkstra handles weights. Uses a min-heap (priority queue) to always process the nearest unvisited node first.

Example — network delay time

import heapq
from collections import defaultdict

def network_delay(times, n, src):
  graph = defaultdict(list)
  for u, v, w in times: graph[u].append((w, v))
  dist = {src: 0}
  heap = [(0, src)]
  while heap:
    d, u = heapq.heappop(heap)
    if d > dist.get(u, float("inf")): continue
    for w, v in graph[u]:
      nd = d + w
      if nd < dist.get(v, float("inf")):
        dist[v] = nd; heapq.heappush(heap, (nd, v))
  return max(dist.values()) if len(dist) == n else -1
Spot it when: weighted graphshortest pathnon-negative weightsmin cost
