Q1 You're storing user profiles and need to retrieve a profile instantly by username. Which structure do you use? ✓

Answer

dict
O(1) lookup

A dict maps each username (key) to its profile (value). Retrieval is O(1) — it doesn't matter if you have 10 or 10 million users.

Code

profiles = {
  "alice": {"age": 30, "city": "Paris"},
  "bob":   {"age": 25, "city": "Lyon"},
}

# O(1) — instant regardless of size
user = profiles["alice"]
# {"age": 30, "city": "Paris"}
Q2 You're building a login system and need to check whether a username is banned. The check runs on every login attempt. ✓

Answer

set
O(1) membership

You only need "is X in there?" — no value needed. A set gives O(1) membership checks. A list would be O(n) — catastrophic at scale.

Code

banned = {"admin", "root", "superuser"}

def can_login(username):
  if username in banned:  # O(1)
    return False
  return True

# list would be O(n) — never do this
# banned_list = ["admin", "root", ...]
Q3 You're processing a pipeline of tasks that must run in a specific order. You need to access them by their step number (0, 1, 2…). ✓

Answer

list
O(1) by index

You need ordered, indexed access. A list is perfect: steps[0] is O(1). A tuple would also work but use list if steps might be added dynamically.

Code

pipeline = ["parse", "validate", "save", "notify"]

for i, step in enumerate(pipeline):
  print(f"Step {i}: {step}")

# Access by index — O(1)
current = pipeline[0]   # "parse"
last    = pipeline[-1]  # "notify"
Q4 You store GPS coordinates (latitude, longitude) for cities. These values never change and you want to use them as dictionary keys. ✓

Answer

tuple
hashable → O(1) key

Only hashable types can be dict keys. Lists are mutable → not hashable → TypeError. Tuples are immutable → hashable → perfect composite key.

Code

cities = {}

# ✅ tuple — hashable, works as key
cities[(48.85, 2.35)]  = "Paris"
cities[(51.50, -0.12)] = "London"

# ❌ list — TypeError!
# cities[[48.85, 2.35]] = "Paris"

lat, lon = (48.85, 2.35)  # unpack

Q5 Given a list of words, count how many times each word appears. You need to look up a word's count in O(1). ✓

Answer

dict / Counter
O(n) build, O(1) lookup

Build a frequency map in one pass. Counter from collections does this automatically and adds most_common(). Both are O(n) to build, O(1) per lookup.

Code

from collections import Counter

words = ["cat", "dog", "cat", "bird", "dog", "cat"]

counts = Counter(words)
# Counter({"cat": 3, "dog": 2, "bird": 1})

counts["cat"]          # 3 — O(1)
counts.most_common(1) # [("cat", 3)]
Q6 You need to group a list of words by their first letter — e.g. all words starting with "a" together, "b" together, etc. ✓

Answer

dict of lists
O(n)

Map each category (key) to a list of items (value). Use defaultdict(list) to avoid checking if the key exists before appending.

Code

from collections import defaultdict

words = ["apple", "ant", "banana", "avocado", "blueberry"]

groups = defaultdict(list)
for w in words:
  groups[w[0]].append(w)

# {"a": ["apple","ant","avocado"],
#  "b": ["banana","blueberry"]}
Q7 Classic interview: given nums = [2, 7, 11, 15] and target = 9, find two indices that sum to target. You need one-pass O(n). ✓

Answer

dict
O(n) — one pass

Store each value's index as you iterate. For each number, check if its complement (target - n) is already in the dict. O(n) vs O(n²) for nested loops — this is exactly what the interviewer tests.

Code

def two_sum(nums, target):
  seen = {}  # value → index
  for i, n in enumerate(nums):
    complement = target - n
    if complement in seen:  # O(1) check
      return [seen[complement], i]
    seen[n] = i
  return []

two_sum([2, 7, 11, 15], 9)  # [0, 1]
Q8 Find the first character in a string that appears only once. E.g. in "leetcode" the answer is "l". ✓

Answer

dict
O(n)

Count each character with a dict, then scan the original string to find the first with count == 1. Two passes, both O(n). A set can't store counts — only presence.

Code

s = "leetcode"

counts = {}
for ch in s:
  counts[ch] = counts.get(ch, 0) + 1

for ch in s:
  if counts[ch] == 1:
    print(ch)  # "l"
    break

Q9 You receive a list of email addresses from a form — some are duplicates. You need to send one email per unique address. ✓

Answer

set
O(n) dedup

Converting to a set drops all duplicates in one step. If you need to preserve insertion order (e.g. send in the order received), use dict.fromkeys() instead.

Code

emails = ["a@x.com", "b@x.com", "a@x.com", "c@x.com"]

# Fast dedup — order not guaranteed
unique = set(emails)

# Preserve order (Python 3.7+)
unique_ordered = list(dict.fromkeys(emails))
# ["a@x.com", "b@x.com", "c@x.com"]

for addr in unique_ordered:
  send_email(addr)
Q10 You have two lists of users: those who completed module A and those who completed module B. Find users who completed both. ✓

Answer

set & (intersection)
O(min(a,b))

A nested loop is O(n²). Set intersection a & b is O(min(len(a), len(b))). Always convert to sets before intersection — the performance difference is massive at scale.

Code

module_a = {"alice", "bob", "clara"}
module_b = {"bob", "dan", "clara"}

# ✅ O(min(a,b)) — set intersection
both = module_a & module_b
# {"bob", "clara"}

# ❌ O(n²) — never do this
# both = [u for u in module_a if u in module_b_list]
Q11 Check if a list contains any duplicate values. Return True if there are duplicates, False otherwise. Must be O(n). ✓

Answer

set
O(n)

If a list has duplicates, its set will be smaller. Compare len(nums) != len(set(nums)) — one line, O(n). Sorting would be O(n log n). A dict works too but a set is cleaner here.

Code

def has_duplicate(nums):
  return len(nums) != len(set(nums))

has_duplicate([1, 2, 3, 4])      # False
has_duplicate([1, 2, 2, 4])      # True

# Early-exit version (stops sooner)
def has_duplicate_v2(nums):
  seen = set()
  for n in nums:
    if n in seen: return True
    seen.add(n)
  return False
Q12 You have a list of integers from 1 to n with one number missing. Find the missing number efficiently (no sorting). ✓

Answer

set
O(n)

Build a set of expected values and subtract the actual values. The result is the missing number. Set difference - is O(n). Alternatively, use math: n*(n+1)//2 - sum(nums) is even simpler.

Code

nums = [1, 2, 4, 5, 6]  # missing 3
n = max(nums)

# Set difference approach
expected = set(range(1, n + 1))
missing = expected - set(nums)
print(missing)  # {3}

# Math trick — even O(n) with less memory
missing2 = n * (n + 1) // 2 - sum(nums)
print(missing2)  # 3

Q13 You're implementing an undo feature in a text editor. Each action is pushed, and "Ctrl+Z" pops the last one. Which structure? ✓

Answer

list (stack)
O(1) push/pop

LIFO = Last In, First Out = stack. A Python list works perfectly: .append() pushes, .pop() pops — both O(1). No need for deque when you only touch one end.

Code

history = []

# User types something
history.append("typed 'hello'")   # push O(1)
history.append("typed ' world'")  # push O(1)

# Ctrl+Z
last_action = history.pop()          # pop O(1)
print(last_action)  # "typed ' world'"

# Stack state: ["typed 'hello'"]
Q14 You're implementing BFS (breadth-first search) on a graph. You need to add nodes to the back and remove from the front — efficiently. ✓

Answer

deque
O(1) both ends

list.pop(0) is O(n) — all elements shift left. deque.popleft() is O(1). For BFS with large graphs, this is the difference between O(n) and O(n²).

Code

from collections import deque

graph = {"A": ["B", "C"], "B": ["D"], "C": [], "D": []}

queue = deque(["A"])
visited = set()

while queue:
  node = queue.popleft()   # O(1) !
  if node not in visited:
    visited.add(node)
    queue.extend(graph[node])
Q15 Given a string, check if it's a valid sequence of brackets: "([{}])" → valid, "([)]" → invalid. ✓

Answer

list (stack) + dict
O(n)

Classic stack problem: push opening brackets, pop and compare when you see a closing bracket. A dict maps each closing bracket to its expected opener for clean comparison.

Code

def is_valid(s):
  stack = []
  pairs = {")": "(", "]": "[", "}": "{"}
  for ch in s:
    if ch in "([{":
      stack.append(ch)
    elif not stack or stack.pop() != pairs[ch]:
      return False
  return not stack

is_valid("([{}])")   # True
is_valid("([)]")     # False
Q16 A function needs to return both the minimum and maximum of a list. How should you return these two values in Python? ✓

Answer

tuple
idiomatic Python

Returning multiple values in Python implicitly creates a tuple. Unpacking with lo, hi = min_max(nums) is clean and Pythonic. A list would work but signals mutability — semantically wrong for a fixed pair.

Code

def min_max(nums):
  return min(nums), max(nums)  # tuple

# Clean unpacking
lo, hi = min_max([3, 1, 9, 4, 5])
# lo = 1, hi = 9

# What Python actually returns:
result = min_max([3, 1, 9])
print(type(result))  # <class 'tuple'>

Q17 This code is slow on large inputs. What single change fixes it?

seen = []
for n in nums:
  if n in seen: return True
  seen.append(n) ✓

Answer

set
O(n) → was O(n²)

if n in seen on a list is O(n) per iteration → O(n²) total. Changing seen to a set makes membership check O(1) → O(n) total. One word change, massive impact.

Code

# ❌ O(n²) — list lookup inside loop
seen = []
for n in nums:
  if n in seen: return True  # O(n)!
  seen.append(n)

# ✅ O(n) — set lookup is O(1)
seen = set()
for n in nums:
  if n in seen: return True  # O(1) ✓
  seen.add(n)
